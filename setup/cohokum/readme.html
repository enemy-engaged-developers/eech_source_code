Game info

There are a number of ways to play this game:

  1. As is, default settings, default graphics. Great game.
  2. Download the superpatch2 with tons of custom made warzones/campaigns/skirmishes
  3. Use a WUT file to change the behaviour of vehicles and aircraft all in the game , increased difficulty (some say more realistic).
  4. Use custom textures to improve the looks of the game.
  5. Browse through the eech.ini and activate for instance "faa=1" (fly any aircraft, jump in returning choppers and have a field day) or for instance "camcom=1", the campaign commander which lets you create missions and give you a greater control of the campaign.
  6. Play online with MP (look on this server to see what's up).

Flyable choppers

As of version 1.6.0.W12 you can fly, change the loadout and repair the following choppers: Comanche, Apache Longbow, HokumB, Havoc, Blackhawk, Hind, AH64-A (Apache), and KA50 (Hokum). The cockpits and the flight behaviour of the new blue choppers default to the Apache Longbow, of the red choppers to the Havok.

Note that you can pick a returning chopper to fly rogue. Spot the wanted Helo in the 2D-map, click on it, accept, auto-select (mission), confirm and off you go (you can't choose it from the mission or group list since it has no task). You can only do this if the option "faa" in eech.ini is set to "1" (see eech.ini).

Jabberwock made any chopper flyable in free-flight mode. Just be sure you activate the "fly any aircraft" switch in eech.ini. Use F12-key to see the aircraft and scroll through the list to the chopper you want to use. Hit F12 again to clear the screen and then hit 'U' and you are in the cockpit of the chosen chopper.

Multiplayer status

Seems to be fairly smooth these days, but the test environment is limited, so there could still be bugs left. It's crucial that the host runs with cvc=1. Its also a good idea to touch only the other variables if you know what you are doing. Downwash should be off, it can mess up the host after a while.

What is a WUT file?

Most people fly with a WUT file. A WUT-profile (Weapons and Units Tweaker file) is a text file that alters some critical parameters in the game to make it more "realistic" and harder. See the wiki on eechcentral For more information about WUT-files.

      Features
    • Added support for generating log files for Tacview analysis. To enable it set tacview_logging in eech.ini to one of the values 1, 2 or 3. 3 will log everything that happens from the moment the campaign starts. 1 and 2 will only start logging once you start a mission, and will only log units which are close to your helicopter. Log mode 1 will also not show enemy units which are not yet discovered, so it will not "spoil" the campaign for you if you don't want to know where all enemy units are. The logs can be found in the directory cohokum/tacview-logs.
      • AI fixes
    • Surface launched missiles will now fire with a slight climb angle to avoid hitting the ground or surrounding vehicles immediately.
    • Fixed bug where SA-19 missiles would often guide towards the completly wrong direction. Combined with the above fix the SA-19 is a much more serious threat now.
    • Fixed bug where aircraft would miss with unguided weapons because they didn't adjust for projectile drop

    • Changed AH-1W to carry TOW on one wing (Hellfire on the other) as this is a common load on real AH-1Ws
    • Changed default weapon load for blackhawk to be just rockets
    • Changed rocket salvo for most aircraft firing rockets to 4 or 8 rockets per salvo (used to be 2). Useful since rockets are much less accurate now, and aircraft flying round and firing two rockets at a time will spend a lot of time over the target.

    • All helicopters now have two hydraulics systems. If only primary is lost controls become more sluggish. If only secondary is lost no immediate effect. If both are lost controls will gradually lock up as pressure is lost (takes 1 to 15 minutes). You are strongly adviced to land before this happens ;-)

    • Added capability for "trinnlaus zoom" for main 3D view, i.e. smoothly changing field of view to any angle (min and max field of view set in eech.ini). A joystick axis (set in eech.ini), or ALT + F3/F4 keys can be used to change zoom.

These are included in gwut190.csv, but note that the GWUT editor (version 1.3) is not yet capable of editing these values, and will delete them from any file which contains them.     • Several changes to guided missile flight model
      • Missiles might now lose guidance if target moves outside seeker's limit. Air-to-air missile will self destruct if this happens, air-to-ground missiles will continue on unguided.
      • Hellfires launched in LOBL mode should now be able to hit targets at close and medium ranges again :)
      • Hellfires launched in LOAL mode now always does the initial climb and hence won't be able to hit targets at close range (closer than 2km).

      • Implemented non-linear axis for cyclic, collective and pedals. Cyclic and pedals are less sensitive around centre, making fine adjustments easier. Collective can be adjusted to have bigger range around 60-100%, where most of the time is spent. Makes hovering easier with better control and finer adjustments.
      • The laser will deactivate when target moves outside of TADS's field of view (or similar targeting system). It will also lose lock on target if target or lasing helicopter makes too violent manouvers for the targeting system to follow. The laser will most likely also disengage when switching targets in manual laser mode.
      • AI wingmen won't any longer need to climb to get line of sight to target when firing radar guided hellfires. They will instead use LOAL mode and fire from behind cover.
      • AI will no longer waste guided air-to-surface missiles. They will not refire a guided missile at the same target until the first missile has missed (or hit but failed to destroy target). Not just fire a new missile every few seconds even if they have several missiles heading for the target already.
      • Fixed crash with A-model Apache, and all other helicopters using its cockpit.
      • Fixed bug which caused trying to throttle engine down before it was started to actually start the engine.

And now all the PFZ commands are at NUMPAD DEL. ALT deselects current PFZ, and CTRL delets it (and SHIFT and plain cycle PFZs like it used to do).

    Fixes/features in 1.7:

    Flight model changes
    1. Numerous changes to make autorotation work (read here to learn how to autorotate).
      • Rotor brake is now actually engaging/disenging rotor to engine, it's not a brake
      • The collective now works also when the rotor is disengaged
      • Landing with engine failures don't automaticly make the helicopter explode
      • Created new aerodynamic model for rotor RPM when rotor is disengaged. Rotor RPM is now affect by autorotation which increases rotor RPM, and induced and profile drag which reduces rotor RPM.

      • When the rotor is disengaged, the main rotor drives the tail rotor so that it doesn't stop.
      • When the rotor is disengaged rotor rpm may exceed 100%, but get it too high and the rotor might fail as well. Add collective to slow the rotor down again.

      • Added modelling of gas turbine part of engine (and not only power turbine, as it used to be)
      • Engine control system automaticly controls gas turbine RPM in an effort to keep rotor RPM at 100%
      • Added abillity to control throttle and starters for engines, including full startup procedure
      • Added option for starting mission with engines off
      • Modelling of APU added
      • Modelling of engine temperature added
      • Engine fire due to overtemperature now happens at random time, probability depending on temperature.
      • Improved engine instruments to show engine information which previously was not modelled.
      • Modelled sounds for turbines and APU to correspond with RPM of turbines
